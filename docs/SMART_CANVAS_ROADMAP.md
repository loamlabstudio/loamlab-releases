# 智能語義畫布 (Smart Semantic Canvas) 交互模塊 Roadmap

這份 Roadmap 詳述了針對 SketchUp 渲染插件的「智能語義雙引擎」交互模塊的發展演進與具體執行計畫，旨在極大化前後端溝通精準度與用戶Wow Factor，實現一鍵局部重繪的頂級體驗。

---

## 🚀 核心願景與第一性原理

本項目的核心理念是：**將「複雜的局部修改需求」轉化為「零摩擦的圖形化交互」。**
用戶不需要手繪不精準的遮罩，也不需要用複雜的語言描述位置，只需利用「智能語義畫布」進行所見即所得的操作。

### 雙引擎架構
1. **魔術棒引擎 (Smart Select)**：基於 SketchUp 後端輸出的「材質/物件通道圖 (Color ID Map)」，實現前端游標懸停高亮、一鍵選取整個物件的功能。
2. **自由畫布引擎 (Freehand Draw)**：提供帶有膠囊狀全像工具列的自定義繪製模式，處理非標準的遮罩需求，並支援直接文字標註。

---

## 🗺️ 開發路線圖 (Phases)

為了確保項目穩步推進且架構解耦，將開發分為以下四個 Phase，逐步達成終極形態。

### Phase 1: 後端基建與通道圖解鎖 (Week 1)
**目標：啟動 SketchUp 端的通道圖生成能力，為智能選取提供數據支撐。**
*   **[ ] 解鎖通道渲染**：修改 SketchUp `loamlab_plugin.rb` 匯出邏輯，在使用者截取 RGB 預覽圖時，觸發第二次「隱藏邊線與陰影、依材質著色 (Color by Material)」的視圖匯出。
*   **[ ] 雙圖聯動傳輸**：確保前端接收到的資料包含 `originalImage (Base64)` 與 `channelImage (Base64)` 兩部分。
*   **[ ] 性能優化**：透過 Ruby API 確保通道圖生成速度極快，不影響用戶現有的出圖體驗，預期時間 < 500ms。

### Phase 2: 前端視網膜級交互組件 (Week 2)
**目標：打造這套系統的 UI 外殼（如圖所示的膠囊工具列）與隱藏畫布架構。**
*   **[ ] `SemanticCanvas` 架構**：
    *   在 `index.html` 構建疊層結構：最底層為原預覽圖 -> 隱藏層為通道圖 -> 頂層為操作 Canvas (`pointer-events: auto`)。
*   **[ ] 膠囊工具列 (Floating Tool Capsule)**：
    *   實作具備毛玻璃效果 (Glassmorphism) 的底部位懸浮控制條。
    *   包含：智慧選取 (魔術棒)、畫筆、多色選擇器、文字工具(T)、橡皮擦、清空(Clear)、復原(Undo)。
*   **[ ] 動態輸入標籤**：
    *   用戶點擊選中區域後，在游標旁彈出具備微動效的（例如 Blur-to-Clear）迷你文字輸入框。

### Phase 3: 核心算法與雙引擎資料流 (Week 3)
**目標：實現滑鼠 Hover 自動高亮與多區域數據封裝。**
*   **[ ] 魔術棒選取算法**：
    *   追蹤游標 `(x, y)` 座標，從隱藏通道圖中提取 `RGB` 陣列值。
    *   使用 WebWorker 或 `requestAnimationFrame` 掃描整張通道圖，找出所有相同的 RGB 像素，並在頂層 Canvas 用半透明純色塗滿，形成「懸停發光」的驚豔效果。
*   **[ ] 多區域狀態管理 (Region Manager)**：
    *   維護一個陣列 `interactions = []`。
    *   記錄每一次的點擊/畫筆與對應的文本。
    *   設計一個將任意形狀轉換為純黑白 `Mask Base64` 的編碼器。

### Phase 4: 後端渲染串接與閉環 (Week 4)
**目標：將精確無誤的多重遮罩與需求發給 AI 模型，實現精準 Inpainting。**
*   **[ ] Payload 封裝**：
    *   修改 `app.js` 送出給後端 API 的數據格式：
      ```json
      {
        "image": "base64_string",
        "inpaints": [
          { "mask": "base64_mask_1", "prompt": "字體改成xxx" },
          { "mask": "base64_mask_2", "prompt": "沙發換成xxx" }
        ]
      }
      ```
*   **[ ] ComfyUI / 後端解析**：
    *   調整後端的渲染工作流（如 Flux 2 Inpainting），使其能接收 List of Masks 並且依照區域進行無縫融合渲染。
*   **[ ] 測試與除錯**：針對邊緣情況（遮罩重疊、無文字 Prompt、顏色容差問題）進行測試修復。

## 📈 未來展望 (Phase 5+)
- **語義化推薦**：當用戶框選到某個通道顏色（例如識別為地板），下方膠囊工具列直接推薦常用的 Prompt（如：溫潤木地板、灰色水磨石）。
- **Segment Anything (SAM) 整合**：若 SketchUp 複雜場景無法導出完美通道圖，可引入前端或邊緣端的極輕量級 SAM 模型，實現真正意義上的「萬物皆可點擊」。
- **版本控制與歷史**：允許用戶保留多種不同材質組合的塗鴉圖層，一鍵預覽切換。
